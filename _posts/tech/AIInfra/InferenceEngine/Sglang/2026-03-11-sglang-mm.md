# SGlang's memory management system
## TL;DR
- Key-Vlue(KV) can be reused in token generation, which benefits the inference process when be catched.
- SGLang uses maps to find the KV cache indices of a tenson stored in the memory pool.
- The map methods can satisfy to multiple attention backends, such as MHA, MLA and NSA.
- There are multiple catche backends to satisfy different using scene, perfomance and implimentation requirements. 
|Cache Class | Module | Condition |
|:-----|-----:|-----:|
|RadixCache | mem_cache/radix_cache.py | Default|
|ChunkCache | mem_cache/chunk_cache.py | disable_radix_cache=True|
|SWAChunkCache |  mem_cache/chunk_cache.py  |  disable_radix_cache=True + sliding window|
|HiRadixCache  |  mem_cache/hiradix_cache.py | enable_hierarchical_cache=True|
|SWARadixCache |  mem_cache/swa_radix_cache.py  |  Sliding-window attention model|
|MambaRadixCache | mem_cache/mamba_radix_cache.py | Mamba/SSM hybrid model|
|LMCRadixCache | mem_cache/storage/lmcache/ | enable_lmcache=True|
|RadixCacheCpp | mem_cache/radix_cache_cpp.py  | Experimental C++ radix tree|
