# SGlang's memory management system
## TL;DR
- Key-Vlue(KV) can be reused in token generation, which benefits the inference process when be catched.
- SGLang uses a two-level memory pool architecture for efficient KV cache management: ReqToTokenPool maps requests to token locations, while TokenToKVPoolAllocator manages KV cache indices with KVCache storing physical data.
- The system supports hierarchical caching (HiCache) with GPU memory as L1, host memory as L2, and distributed storage as L3, dramatically expanding cache capacity for long-context workloads.
